Eine der Sachen, die ich bei Markdown nicht so gut gelungen finde sind Bilder,
insbesondere mit Bildunterschriften.
Dafür gibt es ja seit HTML5 das `figure`-Tag, das zusammen mit `img` und `figcaption`
genau das macht, was Blogger und Designer seit Jahrzehnten nachzubauen versuchen.
Dank der Liquid-Template-Sprache, die ich hier einsetzen kann, muss mich das nicht stören,
ich kann mit einem einfachen Include selber Bilder einbinden
(allerdings vor allem technisch, nicht schön - bin schließlich kein Designer :):

{% include image img='vogel.by.anira..JPG' text='Nice Bird' title="bird" %}


<!-- more -->

(Das Bild habe ich bei [the inspiring live](http://www.the-inspiring-life.com/2013/01/ein-logo-fur-meinen-blog.html) gefunden.)

Und das geht so:

In `\_includes` eine Datei namens `image` erstellen, die folgenden Code enthält:

{% highlight html %}
{% raw  %}
<figure>
  <a href="/assets/{{ include.img }}">
    <span class="rounded-img" style="background: url(/assets/{{ include.img }}) no-repeat center center;">
      <img {% if include.title %}alt="{{ include.title }}" {% endif %}src="/assets/{{ include.img }}" style="opacity: 0;"/>
    </span>
    <figcaption>{{ include.text }}</figcaption>
  </a>
</figure>
{% endraw %}
{% endhighlight %}

Das Bild auf der Seite einbinden:
{% highlight html %}
{% raw  %}
{% include image img="vogel.by.anira..JPG" text="Nice Bird" title="bird" %}
{% endraw %}
{% endhighlight %}

Ein bisschen aufhübschen mit CSS

{% highlight css %}

.rounded-img {
    display: inline-block;
    border: solid 1px #000;
    overflow: hidden;
    -webkit-border-radius: 10px;
    -moz-border-radius: 10px;
    border-radius: 10px;
    -webkit-box-shadow: 0 1px 3px rgba(0, 0, 0, .4);
    -moz-box-shadow: 0 1px 3px rgba(0, 0, 0, .4);
    box-shadow: 0 1px 3px rgba(0, 0, 0, .4);
}
{% endhighlight %}

Fertig. Wenn ich keine Bildunterschrift will, lasse ich das Attribut einfach weg:
{% highlight html %}
{% raw  %}
{% include image img='mug.jpg' title="bird" %}
{% endraw %}
{% endhighlight %}
